# AGENTS.md

## Objetivo del repositorio
Este workspace se usa para investigar noticias, anuncios, artículos y videos de tecnología y convertirlos en informes editoriales originales para artículos de blog.

## Reglas editoriales globales
- Priorizar fuentes primarias cuando existan; usar cobertura secundaria para contexto, no como autoridad final.
- No hacer copy-paste de las fuentes, salvo citas cortas y atribuidas cuando sean necesarias.
- Separar siempre cada afirmación en una de estas categorías: hecho verificado, inferencia, hipótesis o punto no resuelto.
- No presentar opinión, marketing o especulación como si fueran hechos.
- Cuando una afirmación dependa de información reciente, verificarla con fuentes actuales y usar fechas exactas.
- Conservar enlaces directos a todas las fuentes relevantes.
- Tratar videos de YouTube, shorts y clips sociales como pistas a verificar, no como verdad autosuficiente.
- Señalar contradicciones, omisiones e incentivos del emisor cuando aporten contexto.
- Ajustar el idioma de salida al idioma del pedido del usuario.

## Estándar de salida
- Entregar primero un informe de investigación estructurado, no un borrador de artículo, salvo que el usuario pida el artículo.
- Incluir resumen ejecutivo, evaluación de la fuente, hechos verificados, interpretación, contexto relacionado, hipótesis, implicaciones y explicación en lenguaje claro.
- Incluir una mirada experta y una versión comprensible para público general cuando aporte valor.
- Si el usuario pide el artículo final, partir del headline elegido, la audiencia y la tesis para escribir una pieza original, no una réplica del reporte ni de la fuente.
- Etiquetar incertidumbre y nivel de confianza cuando falte evidencia.
- Evitar tono marketinero, grandilocuente o sensacionalista.

## Contexto editorial
- Priorizar inteligencia artificial, producto, plataformas, desarrolladores, infraestructura, tooling y adopción tecnológica.
- Explicar impacto para negocio, operaciones, implementación y adopción real.
- Considerar implicaciones para LATAM cuando sea relevante.

## Convención de almacenamiento
- Guardar informes terminados en `research/reports/`.
- Usar el patrón `YYYY-MM-DD-topic-slug.md` salvo que el usuario pida otro nombre.
